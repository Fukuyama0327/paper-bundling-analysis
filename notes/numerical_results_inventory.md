### 期待契約件数関数

中間審査資料では以下の式として整理されている。

```text
f(N) = 1.594N / (128.7 + N)
R^2 = 0.997
```

この式は査読論文化で重要な中核結果になる。元データ・生成コード・採用シナリオを確認する必要がある。

追加確認:
- `図表作成/output/combined_scenarios_datav2.xlsx` の `Fitting_Results` を確認したところ、この式は `l=5` シナリオの `fraction` モデルに対応する。
- 復元値は `Param_a = 1.594364940998926`, `Param_b = 128.7141831342621`, `R^2 = 0.9974120332290922`。
- 詳細は `notes/figure2_fit_recovery.md` を参照。
- `parallel_contracts.log` から、`l=5` のシミュレーション条件は `T=100`, `S=10000`, 初期健全度分布 `[0.3250774, 0.6501548, 0.0247678]` と確認した。

後続更新:
- `20251126_中里さんから受領` に、期待契約件数の解析解PDF、比較ノートブック、比較CSVが存在する。
- そこでは、シミュレーション結果のフィッティングではなく、単一橋梁の補修確率 \(q\) と二項分布に基づく閉形式の期待契約件数関数を使っている。
- 詳細は `notes/closed_form_expected_contracts.md` を参照。
- 閉形式比較で用いた推移確率行列はノートブック出力から復元済み。そこから得られる単一橋梁の補修確率は `q = 0.012320`。

### 地域分割最適化結果

正本側の `図表作成/最適解v2.csv` には、以下の目的関数値が含まれていた。

| 条件 | ObjectiveValue |
|---|---:|
| 既存の管理 | 2.5241 |
| D=15km, M=6 | 2.0757 |
| D=20km, M=4 | 1.8086 |
| D=25km, M=3 | 1.5443 |
| D=30km, M=3 | 1.3848 |
| D=35km, M=3 | 1.2152 |
| D=40km, M=1 | 1.1390 |
| D=45km, M=1 | 1.1390 |
| D=50km, M=1 | 1.1390 |

読み取り:
- 距離制約を緩和すると、目的関数値は概ね低下。
- 既存管理の 2.5241 に対して、D=25km, M=3 では 1.5443、D=35km, M=3 では 1.2152。
- D=40km以上ではM=1が可能になり、目的関数値は1.1390。

注意:
- これらの値は正本側の結果であり、査読論文化で採用するなら再生成可能性を確認する。
- `20251208_定期打ち合わせ` 側の `analysis_report_simple_D25km_20251207_200314.txt` には Objective=3.969977 の別結果がある。設定または目的関数が異なる可能性が高いため、現時点では正本docxの直接更新には使わない。

## 設定ファイルから確認した値

`20251208_定期打ち合わせ/config/config.py` 由来の初期移植コードでは、以下が設定されている。

```text
T = 100
I = 5
S = 10000
M_RANGE = (1, 7)
MARKOV_TRANSITION_MATRIX =
  [0.95, 0.05, 0.00]
  [0.00, 0.94, 0.06]
  [0.00, 0.00, 1.00]
```

注意:
- 正本docxの推定推移確率行列とは異なる可能性がある。
- 査読論文では、論文で採用する分析結果とコード設定を一致させる必要がある。
- 同時発注上限 `I = 5` は、図2の `l=5` フィッティング結果と整合する可能性が高い。
- ただし、図2復元元のログでは `N_values=323`、`initial_conditions.csv` では対象橋梁数322であり、両者の意味の違いを本文・コード整理時に確認する必要がある。
